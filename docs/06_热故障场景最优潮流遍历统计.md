# 热故障 + 源荷失衡场景下 DC-OPF 遍历与切负荷统计

本文档对应 `verify/fault_scenarios_2000.csv` 中的 2000 个蒙特卡洛故障场景。每个场景为 85 维 0/1 向量：

\[
[G_1,\ldots,G_{10}, L_1,\ldots,L_{46}, T_1,\ldots,T_{29}]
\]

其中 0 表示故障，1 表示良好。`T1..T29` 对应 IEEE39 修改算例中所有非电源节点的升序列表，即母线 1--29。

## 1. 故障场景到 OPF 参数的映射

对第 \(s\) 个场景，令发电机、线路、节点变压器状态分别为
\[
z^G_{s,g},\quad z^L_{s,l},\quad z^T_{s,t}\in\{0,1\}.
\]

### 1.1 发电机故障

若 \(z^G_{s,g}=0\)，则该机组故障，最大出力置零。实现中同时将调度下界置零，避免原最小技术出力大于零时出现下界超过上界：

\[
\underline P_{g,s}=0,\qquad \overline P_{g,s}=0.
\]

健康机组仍采用极热无风场景下的原调度上下界。

### 1.2 线路故障

若 \(z^L_{s,l}=0\)，则第 \(l\) 条支路退出 DC 网络，其潮流记为 0。实现时不再把该支路电纳加入 \(B_{\mathrm{bus}}\)，也不再对该支路建立潮流约束：

\[
f_{l,s}=0.
\]

说明：用户描述为“线路潮流为零（即限额为零）”。在 DC-OPF 中，如果保留支路电纳但把限额设为 0，会强制两端相角相等，等价于仍保持电气连接；这不符合线路退出的物理含义。因此代码采用“支路退出 + 报告潮流为 0”的实现。

### 1.3 节点变压器故障

若节点变压器 \(T_t\) 故障，其所在非电源节点为 \(b_t\)，则所有与该节点相连的支路均退出：

\[
a_{s,l}=z^L_{s,l}\cdot \mathbf{1}(f_l\not=b_t)\cdot \mathbf{1}(t_l\not=b_t),
\]

其中 \(a_{s,l}\) 是最终用于 OPF 的支路可用性。若该节点有负荷，节点功率平衡会强制该节点负荷由切负荷变量吸收，因此体现“变压器所在节点功率为零”。

## 2. 故障后 DC-OPF

每个故障场景均代入第二步 DC-OPF：

\[
\min \sum_g(c_{2,g}P_{g,s}^2+c_{1,g}P_{g,s})
    +\sum_{i,k}\mathrm{VOLL}_k\,\Delta D_{i,k,s}
\]

约束仍包括节点功率平衡、机组调度上下界、切负荷上下界、平衡节点相角，以及健康支路潮流限额。区别是 \(B_{\mathrm{bus}}\)、机组上下界和支路集合均随故障场景变化。

若原调度约束下某场景不可行（例如故障造成含最小技术出力机组的孤岛，但该孤岛无足够负荷吸收），该场景状态记录为 `infeasible`，不混入切负荷分布的分位数统计。

```bash
cd verify
python3 evaluate_fault_scenarios.py
```

```matlab
cd matlab
stats = evaluate_fault_scenarios();
```

## 4. 2000 个场景的切负荷统计

基准极热无风、无元件故障时：

\[
D_{\mathrm{total}}=6754.5684\ \mathrm{MW},\qquad
\Delta D_0=762.8059\ \mathrm{MW}.
\]

2000 个故障场景中：

| 状态 | 场景数 | 比例 |
|---|---:|---:|
| `optimal` | 1976 | 98.8% |
| `infeasible` | 24 | 1.2% |

| 最小值 | 762.806 | 0.000 |
| 平均值 | 940.292 | 177.487 |
| 标准差 | 365.466 | 365.466 |
| 5% 分位 | 762.806 | 0.000 |
| 25% 分位 | 762.806 | 0.000 |
| 中位数 | 762.806 | 0.000 |

分等级切负荷统计（可解场景）：

| 负荷等级 | 平均切负荷 MW | 95% 分位 MW | 最大值 MW | 出现正切负荷的样本比例 |

总切负荷最大的场景主要由多台大容量火电机组故障造成：

| 场景 | 总切负荷 MW | 切负荷率 | 主要故障 |

| 1792 | 3330.397 | 49.31% | G2, G4, G10 |

| 1752 | 2667.820 | 39.50% | G2, G10 |
| 1234 | 2643.056 | 39.13% | G9, G10 |
| 1479 | 2643.056 | 39.13% | G9, G10 |

不可行场景通常与网络割裂或含最小技术出力机组的孤岛有关。例如 L46、L14、L33 等支路故障可造成局部孤岛；若孤岛内机组受原调度下界约束而无法平衡，则原模型判定不可行。
