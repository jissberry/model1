# 第五步　基于元件故障概率的蒙特卡洛故障场景抽样

本节在 `docs/04_极热条件元件故障概率模型.md` 得到的元件故障概率基础上，生成极热场景下的
蒙特卡洛故障场景集合。每个故障场景用一个 **85 维 0/1 向量**表示：

$$
\mathbf{z}^{(s)}=
\big[
G_1,\ldots,G_{10},\;
L_1,\ldots,L_{46},\;
T_1,\ldots,T_{29}
\big]\in\{0,1\}^{85}
$$

其中：

- $0$：该元件在本次抽样中**故障**；
- $1$：该元件在本次抽样中**良好**；
- $s=1,\ldots,2000$ 为场景编号。

---

## 1. 向量顺序

| 分块 | 维度位置（1-based） | 元件 |
|---|---:|---|
| 发电机 | 1–10 | $G_1,\ldots,G_{10}$ |
| 线路/支路 | 11–56 | $L_1,\ldots,L_{46}$ |
| 节点变压器 | 57–85 | $T_1,\ldots,T_{29}$ |

因此总维数：

$$
10+46+29=85
$$

> 注：线路分块包含 IEEE 39 节点算例的全部 46 条支路；其中 `tap≠0` 的 12 条支路仍作为线路/支路元件参与抽样，**不作为变压器元件**。变压器元件按 `docs/04` 的假设为 29 台节点变压器（每个非电源节点一台）。

---

## 2. 抽样规则

设第 $i$ 个元件的故障概率为 $p_i$。对每个场景、每个元件独立产生随机数：

$$
u_i^{(s)}\sim U(0,1)
$$

并按下式得到状态：

$$
z_i^{(s)}=
\begin{cases}
0, & u_i^{(s)} < p_i \quad(\text{故障})\\
1, & u_i^{(s)} \ge p_i \quad(\text{良好})
\end{cases}
$$

例如：若某元件 $p_i=0.8$，则一次抽样中它有 $80\%$ 概率取 $0$，有 $20\%$ 概率取 $1$。

本算例使用固定随机种子：

$$
\mathrm{seed}=20260627
$$

保证结果可复现。

## 5. 运行方式

### MATLAB

```matlab
cd matlab

% 自动计算故障概率并生成 2000 个 85 维场景
mc = monte_carlo_fault_scenarios();

% 指定场景数、随机种子、CSV 输出
fp = fault_probability();  % 若已传入 res/mpc/sc，可用 fault_probability(res,mpc,sc,false)
mc = monte_carlo_fault_scenarios(fp, 2000, 20260627, '../verify/fault_scenarios_2000_from_matlab.csv');
```

### Python

```bash
cd verify
python3 mc_fault_scenarios.py
```

输出：

```text
generated 2000 scenarios x 85 components
vector order: [G1..G10, L1..L46, T1..T29]
encoding: 0=fault, 1=healthy
```

## 6. 后续用途

这 2000 个故障场景可作为后续可靠性/韧性评估输入，例如：

1. 对每个场景移除故障元件（发电机、线路、节点变压器）；
2. 重新进行潮流/最优潮流或连通性判别；
3. 统计失负荷、电源损失、线路断面风险、节点供电可达性等指标；
4. 得到极热条件下系统风险的经验分布。
